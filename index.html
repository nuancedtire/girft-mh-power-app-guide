01

Before the first click

Build a local improvement workflow, not another spreadsheet.

The app records one eligible 12+ hour mental-health-related ED stay per case. It supports the immediate GIRFT return of 10 consecutive patients and can continue as a local quality-improvement dataset after the collection period.

OWNER

Nominate three roles

Contributor: creates and edits cases. Reviewer: quality-checks and approves. Owner: manages permissions, app changes, exports and retention.

DATA BOUNDARY

Keep it non-identifiable

No name, NHS number, date of birth, address, free-text chronology or other information that could identify a patient. Use only Case 01 to Case 10 or a locally agreed anonymous reference.

Recommended first release: SharePoint List as the data store, Canvas App as the guided entry interface, Power Automate only for notifications/PDF if approved, and Power BI only after the initial 10 cases are complete.

02

SharePoint

Create the data store first.

  1. Open the approved Trust/team SharePoint site.
  2. Select New → List → Blank list.
  3. Name it GIRFT MH ED Breach Cases. Do not make it publicly visible.
  4. In List settings, turn off attachments unless a local IG lead expressly requires them. This workflow should not accept clinical documents.
  5. Keep version history enabled. In Advanced settings, allow users to edit only items they created if contributors should not see each other's cases.

03

Power Apps

Make entry feel like a guided review.

  1. Go to make.powerapps.com in the correct NHS environment.
  2. Select Create → Blank canvas app → Tablet. Name it GIRFT MH ED Breach Collection.
  3. Select Data → Add data → SharePoint, then connect the GIRFT MH ED Breach Cases list.
  4. Create four screens: Dashboard, Case entry, Review, and Submission view.
01

Dashboard

Ten numbered case cards, workflow status, missing-field count, and “new case” button.

02

Case entry

Five collapsible sections: Overview, Demographics, Chronology, Safety, Reflection.

03

Review

Read-only summary with a completeness check and approve/return actions.

04

Submission

Answers in the same sequence as the NHS online form, ready for manual transcription.

Essential Power Fx

Use a form named frmCase, a collection named colMissing, and a gallery named galCases. Adapt column names to your final SharePoint internal names.

// Create a draft case
NewForm(frmCase);
Navigate(scrCaseEntry, ScreenTransition.Cover)

// Save without completing the return
SubmitForm(frmCase);

// Save and mark ready for review
If(
    CountRows(colMissing) = 0,
    Patch('GIRFT MH ED Breach Cases', frmCase.LastSubmit,
        {'Workflow status': {Value: "Ready for review"}}),
    Notify("Complete the highlighted required fields before review.", NotificationType.Error)
)

// Conditional section: escalation questions
ddEscalated.Selected.Value = "Yes"

// Conditional section: MHLT chronology
ddMHLTReferral.Selected.Value = "Yes"
Important: SharePoint multi-choice fields require the Combo box’s selected records, not a concatenated string. In an Edit Form data card, use the generated Update formula and do not overwrite it with text.
04

Quality and access

Make the right thing the easy thing.

PERMISSIONS

Keep roles deliberate

  • Owners: site/list and app administration.
  • Contributors: create and edit their cases.
  • Reviewers: read all cases, approve or return them.
  • Do not use anonymous sharing links.
QUALITY GATE

Review before NHS entry

  • Check eligibility: MH-related attendance and ED stay >12 hours.
  • Check 10 cases are consecutive.
  • Check no identifiable text has been entered.
  • Use “Not recorded” where permitted instead of an empty required answer.

Test with synthetic data only

  1. A contributor can create, save and resume a draft.
  2. Multi-select answers save and reopen correctly.
  3. “No escalation” hides Q13–Q16.
  4. “No MHLT referral” hides MHLT-specific chronology but retains safety/outcome questions.
  5. A reviewer can return a case with a non-identifying note.
  6. The submission view uses the selected case and displays all expected answers.
  7. A mobile/tablet user can complete a long chronology without horizontal scrolling.
05

Handover and learning

Export is a workflow, not a button.

1ApproveReviewer checks each case.
2SubmitAdministrator transcribes to NHS Citizen Space.
3RecordSet workflow status to Submitted to NHS.
4ExportList → Export to Excel for approved local analysis.

Three safe outputs

  • NHS return: manually transcribe the approved case using the Submission screen. Citizen Space does not provide a standard direct integration.
  • Local spreadsheet: in the SharePoint List, select the view, filter to Approved or Submitted to NHS, then use Export → Export to Excel.
  • Dashboard: connect Power BI to the List only after data-quality checks. Show aggregated delay drivers and timings; avoid case-level free-text in broadly shared reports.
Retention: agree who owns the List, who may export it, and when it will be deleted or archived under local information-governance rules before collecting live data.
